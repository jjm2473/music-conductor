# Music Conductor PRD

## 1. 项目目标
Music Conductor 是一个面向个人用户的本地音乐文件管理器，采用 B/S 架构，提供以下核心能力：
1. 文件名特殊字符替换
2. 去重
3. 批量重命名
4. 元数据编辑
5. 批量元数据填充
6. 批量元数据清理
7. 现代化界面

目标场景为个人音乐库管理（几百到上千首规模），优先保证流程清晰、可确认、可追踪。

## 2. 范围与非目标

### 2.1 范围
1. 仅处理用户选择的单个目录中的音乐文件。
2. 支持音乐文件与同名 lrc 文件联动处理。
3. 提供试听、列表筛选、批量操作、操作清单确认。

### 2.2 非目标
1. 不扫描子目录。
2. 不处理隐藏文件。
3. 不跟随软链接。
4. 删除后不提供恢复机制。
5. 暂不支持超过两种文件名模式的智能识别（仅支持“艺术家 - 歌曲名”与“歌曲名 - 艺术家”）。

## 3. 系统与启动

### 3.1 架构
1. 前端：Web UI（现代化交互）。
2. 后端：本地服务进程，负责文件扫描、重命名、删除、元数据读写。

### 3.2 启动行为
1. 若服务端启动参数提供音乐目录，直接使用。
2. 若未提供，启动后要求用户选择目录。
3. 若目录不可访问，界面阻止进入主流程并提示错误。

### 3.3 安全策略
1. 默认监听 `127.0.0.1`。
2. 用户可通过配置或参数指定其他监听 IP。
3. 默认无需登录。
4. 若配置启用登录，则所有文件操作接口必须鉴权后可用。

## 4. 扫描与文件范围规则

### 4.1 扫描范围
1. 仅扫描用户选择目录的第一层文件。
2. 忽略目录项、隐藏文件、软链接。

### 4.2 音乐文件识别
1. 扫描与列表仅包含音乐扩展名集合命中的文件。
2. 默认扩展名建议：`mp3, flac, ogg, opus, m4a, mp4, aac, wma, ape, wav, aiff`。
3. 扩展名集合可配置。

### 4.3 lrc 联动
1. 当音乐文件发生重命名时，若同名 `.lrc` 存在，需同步重命名。
2. 去重流程中需处理 lrc 迁移与清理（见第 6 章）。

## 5. 文件名特殊字符替换

### 5.1 输入
1. 可配置映射表（字符或字符串映射）。
2. 对命中映射规则的文件生成重命名计划。

### 5.2 交互
1. 展示变更清单（原名 -> 新名，含 lrc 变更）。
2. 用户确认后执行。

### 5.3 冲突处理
1. 若存在任一重命名冲突，确认按钮置灰。
2. 冲突项在清单中标记并提示原因。

冲突定义：
1. 目标文件名已存在，且不是当前源文件。
2. 多个源文件映射到同一个目标文件名。
3. lrc 目标名冲突（规则同上）。

## 6. 去重

### 6.1 重复定义
重复文件按“文件名（不含扩展名）+ 双向等价”定义，不看元数据。

规则：
1. 仅识别 `A - B.xxx` 格式。
2. `A - B.xxx` 与 `A - B.yyy` 视为重复。
3. `A - B.xxx` 与 `B - A.yyy` 视为重复。

示例：
1. `周杰伦 - 晴天.mp3` 与 `周杰伦 - 晴天.ogg` 重复。
2. `周杰伦 - 晴天.mp3` 与 `晴天 - 周杰伦.flac` 重复。

### 6.2 `.mcignore`
1. 存放于当前音乐目录下。
2. 记录被忽略组中的“原始文件名（含扩展名）”。
3. 一组重复文件若选择忽略，该组所有文件名都写入 `.mcignore`。
4. 下次去重扫描时，命中 `.mcignore` 的文件不参与分组。

### 6.3 交互流程
1. 弹窗按“每组一行”展示重复组。
2. 每组支持试听。
3. 每组支持：
	- 勾选保留文件
	- 勾选“忽略这一组”
4. 每组默认选中“忽略这一组”。
5. 用户确认后，先生成删除清单，再次确认后执行删除。

### 6.4 删除与 lrc 处理
1. 删除仅在用户确认清单后执行。
2. 删除后不提供恢复机制。
3. 若保留文件缺少同名 lrc，且组内其他文件存在可用 lrc，则重命名迁移一份到保留文件名。
4. 组内未使用的其他 lrc 删除。
5. lrc 的重命名/删除操作需展示清单，但无需额外确认。

## 7. 管理器列表

### 7.1 列表字段
1. 文件名
2. 格式
3. 大小
4. 修改时间
5. 时长
6. 元数据：歌曲名、艺术家、专辑

### 7.2 列表能力
1. 支持排序。
2. 支持过滤。
3. 支持滑选。
4. 选中后按空格切换复选框状态。
5. 每行支持打开完整元数据详情。
6. 每行支持编辑常用元数据。

### 7.3 时间滑杆
横向时间滑杆，用于过滤“最近修改”。

刻度定义：
1. 1 小时内：按分钟
2. 1 天内：按小时
3. 1 周内：按天
4. 1 个月内：按周
5. 1 年内：按月
6. 超过 1 年：2 年、5 年、10 年、20 年、所有

过滤语义：展示“修改时间 >= 当前时间 - 选中时间跨度”的文件。

## 8. 批量操作

### 8.1 选择操作
1. 全选
2. 反选
3. 全部取消选择

### 8.2 批量重命名
1. 按 `A - B` 变为 `B - A`。
2. 同步处理同名 lrc。
3. 执行前展示变更清单。
4. 若存在冲突，确认按钮置灰并提示。

### 8.3 批量根据文件名填充元数据
支持两种模式：
1. `艺术家 - 歌曲名.xxx`
2. `歌曲名 - 艺术家.xxx`

执行规则：
1. 用户先选择模式。
2. 预览每个文件将写入的字段。
3. 用户确认后执行。

### 8.4 批量根据元数据修改文件名
支持两种模式：
1. `艺术家 - 歌曲名.xxx`
2. `歌曲名 - 艺术家.xxx`

执行规则：
1. 用户先选择模式。
2. 预览重命名清单，如果文件名跟原来一样就忽略。如果部分文件缺少元数据或对应值为空，则提示错误，不加入清单。
3. 如果清单不为空，用户确认后执行。如果清单为空就置灰确认按钮。

### 8.5 批量清理元数据
支持：
1. 删除元数据中匹配的特定字符串（字面量或正则）。
2. 清空指定元数据字段。

执行前必须展示变更清单并确认。

### 8.6 批量修复扩展名
根据识别到的格式修复扩展名。
特别的，对于mp4格式，扩展名使用“.m4a”（因为我们只管音乐文件），aac裸流则用“.aac”。
执行前展示变更清单并确认。

## 9. 元数据支持矩阵

### 9.1 通用规则
1. 优先按文件格式对应标准读取/写入。
2. 若元数据读取失败，可忽略该文件的元数据展示，不影响其他功能。
3. 若写入失败，记录失败并跳过该文件。
4. 当元数据缺失或读不出时，可从文件名按第 8.3 节模式提取候选值。

### 9.2 格式与标签标准
1. MP3：ID3v2（主），可兼容读取 ID3v1。
2. FLAC：Vorbis Comment（含封面块）。
3. OGG/Vorbis：Vorbis Comment。
4. Opus：OpusTags（Vorbis 风格键值）。
5. M4A/MP4：iTunes/MP4 atoms（常用字段）。
6. WMA：ASF 标签。
7. APE/WavPack：APEv2。
8. WAV/AIFF：尽力读取常见标签；写入能力受库支持限制，失败则跳过并记录。

## 10. 长任务执行模型

适用任务：扫描、批量重命名、批量元数据写入、去重执行。

统一规则：
1. 显示进度条（总体进度 + 当前子任务）。
2. 任务期间阻止用户交互界面。
3. 不支持取消。
4. 子任务失败时记录并跳过，继续后续子任务。
5. 任务进行中展示失败子任务列表（可折叠）。
6. 任务结束后展示完整失败清单与原因。

## 11. 配置设计

### 11.1 配置目录与优先级
1. 默认配置目录：`~/.config/music-conductor/`
2. 覆盖方式：服务端参数或环境变量。
3. 生效优先级：参数 > 环境变量 > 配置文件 > 默认值。

### 11.2 建议配置项
1. `server.host`：默认 `127.0.0.1`
2. `server.port`：服务端口
3. `security.enabled`：是否启用登录鉴权（默认 `false`）
4. `scan.musicExtensions`：音乐扩展名集合
5. `filename.specialCharMap`：特殊字符映射表
6. `filename.delimiter`：默认 `" - "`
7. `metadata.filenameParseModes`：默认两种模式

### 11.3 `.mcignore`
1. 文件名固定 `.mcignore`
2. 位置：当前音乐目录
3. 内容：每行一个原始文件名（含扩展名）

## 12. UI 与交互要求

1. 现代化视觉风格，信息层次清晰。
2. 列表、弹窗、清单均支持键盘操作。
3. 清单页面对危险操作（删除）使用高亮提醒。
4. 冲突项使用显著状态样式，且阻止确认。

## 13. 验收标准

### 13.1 功能验收
1. 可正确扫描目标目录第一层音乐文件，且不包含子目录/隐藏文件/软链接。
2. 特殊字符替换可预览、可确认、可联动 lrc。
3. 去重能识别 `A - B` 与 `B - A`，支持试听、保留、忽略、删除确认。
4. `.mcignore` 可在下次去重时生效。
5. 管理器支持排序、过滤、时间滑杆过滤、滑选与空格切换。
6. 批量重命名/填充/清理均有预览清单与确认。
7. 发现冲突时确认按钮置灰。
8. 长任务期间有进度、阻塞交互、失败记录与结束报告。

### 13.2 质量验收
1. 在 1000 首以内文件规模下流程可用、交互可接受。
2. 失败子任务不会阻断整体任务完成。
3. 关键操作均有可追踪结果（成功/失败/原因）。

## 14. 术语定义

1. 重复文件：满足第 6.1 节规则的一组音乐文件。
2. 变更清单：执行前展示的原值与目标值列表。
3. 子任务：单个文件上的一次原子操作（重命名、删除、标签写入等）。
